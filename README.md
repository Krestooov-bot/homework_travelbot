# ✈️ TravelBot: AI Планувальник подорожей

## 📖 Опис проєкту
Цей проєкт реалізовано в рамках завдання "Варіант 2. Планувальник подорожей «TravelBot»". Це інтерактивний веб-застосунок для планування подорожей, який допомагає користувачам створювати маршрути, автоматично підраховувати бюджет та зберігати список місць для відвідування.

## 🚀 Інструкція з запуску
1. Встановіть необхідні залежності з файлу `requirements.txt`:
   ```bash
   pip install -r requirements.txt
2. Створіть файл `.streamlit/` та додайте туди файл з текстом `GOOGLE_API_KEY = "Ваш_ключ"`:
   ```bash
   secrets.toml

## 📂 Опис структури файлів проєкту
Проєкт має чітку модульну структуру, що розділяє клієнтську візуальну частину та внутрішню інтелектуальну логіку ШІ-агента:

`app.py` — головний виконуваний файл веб-застосунку. Він повністю відповідає за розгортання графічного інтерфейсу користувача (UI) через Streamlit, керує глобальним станом поточної сесії (st.session_state), відстежує історію листування, забезпечує стрімінг відповідей моделі у звичайному режимі чату та координує роботу з табами й колонками.
`agent.py` — ізольований програмний модуль ШІ-агента. Містить опис логічного стану графа архітектури LangGraph, підключає мовну модель Gemini та описує функціональні інструменти (Tools), до яких агент може звертатися під час аналізу текстових намірів користувача.
`requirements.txt` — текстовий конфігураційний файл, який містить чіткий перелік та версії сторонніх Python-бібліотек (таких як streamlit, google-genai, langgraph, pandas), необхідних для швидкого розгортання проєкту на будь-якій машині чи хмарному хостингу.
`.gitignore` — інструкція для системи контролю версій Git. Захищає проєкт від випадкового витоку конфіденційних даних, забороняючи відстежувати та публікувати на GitHub папку з локальними ключами доступу .streamlit/secrets.toml та тимчасові файли кешу системи.

## ✨ Реалізовані функції та UI-компоненти
Графічний інтерфейс застосунку повністю відповідає критеріям інструкції та включає наступні особливості реалізації UI:

## 📱 Особливості планування та геометрії інтерфейсу:
Головна область (Двоколонковий layout через `st.columns`): Робочий простір розділено на дві функціональні зони. Ліва колонка відведена під інтерактивний чат-інтерфейс обговорення деталей майбутньої подорожі. Права колонка виступає в ролі інформативної "картки маршруту", де відображаються підсумкові дані та бюджет.

Вкладки інтерфейсу (`st.tabs`): Дозволяють зручно перемикатися між екраном активного планування (чат та метрики використання моделі) та екраном фінального перегляду маршруту і його експорту.

Бічна панель (`st.sidebar`): Використовується для глобального налаштування застосунку. Вона містить радіокнопку для перемикання між режимами роботи (Звичайний чат ↔ Агент з інструментами), повзунки налаштування креативності моделі (температура), блок редагування системного промпта, а також кнопку скидання та очищення поточної історії.

## 🧩 Спеціальні компоненти Streamlit:
`st.form()` (Форма швидкого додавання зупинки): Інтегрована в нижню частину екрана та в sidebar. Вона дозволяє користувачу миттєво вносити нові точки маршруту вручну в обхід текстового аналізу мовною моделлю, групуючи поля введення міста, активності та кількості днів.

`st.metric()` (Компоненти аналітичних метрик): Наочно демонструють ключові цифри подорожі праворуч від чату. Вони відображають загальний розрахований бюджет мандрівки, сумарну кількість запланованих днів у дорозі, а також статистику сесії (кількість повідомлень та використаних токенів).

`st.dataframe()` (Динамічна таблиця маршруту): Виводить структурований список усіх підтверджених зупинок. Стовпці таблиці (Місто, Активність, Днів) автоматично оновлюються як після ручного додавання через форму, так і після команди асистенту в чаті.

`st.download_button()` (Блок експорту даних): Реалізований у вигляді кнопок для скачування інформації. Дозволяє користувачу вивантажити сформовану таблицю плану подорожі у форматі CSV або зберегти повний текстовий лог діалогу у форматі JSON.

## 🛠 Інструменти ШІ-агента (Tools):
`add_stop(city, activity, days)` — програмна функція, яка викликається агентом у разі виявлення наміру користувача додати нове місце. Вона записує дані безпосередньо у стан таблиці.

`estimate_budget(days, daily_budget)` — математичний інструмент для автоматичного калькулювання фінансових витрат на основі тривалості зупинок.

`show_itinerary()` — внутрішній запит агента, який дозволяє йому проаналізувати вже сформований список міст та побудувати на його основі логічні поради чи зв'язні відповіді.

`wikipedia_search(city)` — інструмент інтернет-пошуку для збору цікавих фактів, визначних пам'яток та актуальної інформації про обране місто.
